🛡️ Phase 7: Security Hardening Verification (added April 25-26)
Confirms the Phase 1 P0/P1 security migration is live and intact. Run these after every deployment that touches auth, rules, CSP, or the Firebase config.
  • Login modal shows "אימייל" not "שם משתמש" Migrated
    Click 🔑 / כניסה in the nav.
    Field label is "אימייל" (Email). The plaintext-passwords version had "שם משתמש" (Username) — if you see that, the new code isn't deployed.

    Log in with a Firebase Auth admin account (e.g. gilinaveh@gmail.com).

  • Non-admin email is rejected with a friendly message
    Try logging in with an email that's in Firebase Auth but NOT in /admins/{uid}.
    Toast says "this account is not registered as an admin". You're auto-signed-out. No admin bar.
  • XSS payload renders as text, not code CriticalAuto ✓
    In the comments form, post a comment with text: <img src=x onerror=alert(1)>
    The comment displays the literal string. No alert() pops up. No JS executes.
    (Automated: tests/xss.spec.js covers this against escapeHtml() directly + as a render-time integration test.)
  • Anonymous visitors can still post candles & comments
    Open the site in an incognito window — make sure you're NOT logged in. Light a candle and post a comment.
    Both succeed. Confirms the anonymous-sign-in fallback works under the new Firestore rules.
  • Admin can delete a comment / story

  • No CSP "Refused to load" errors during normal use

    No Content-Security-Policy: …Refused to load… errors. Some www.google.com/sorry warnings under heavy photo load are expected (Drive rate limiting — separate issue, tracked in P1-5/P1-6).

🖼️ Phase 8: Lazy-load Gallery (added April 26)
Confirms the shimmer-skeleton + fade-in behaviour from P1-6. The gallery shouldn't fire image requests for off-screen photos until you scroll near them.
  • Skeleton shimmer is visible while photos load
    Hard-refresh the site, scroll quickly to the photos section, watch the grid cells before each image appears.
    Each empty cell shows a soft gold-tinted gradient shimmer animation that loops smoothly. Once the photo decodes, the shimmer disappears and the photo fades in.
  • Photos fade in instead of popping
    DevTools → Network → throttle to "Slow 3G" → reload → scroll to gallery slowly.
    As each image arrives, it transitions from invisible to visible over ~450ms. No jarring pops.
  • Off-screen photos don't fire requests until scrolled near
    DevTools → Network tab → reload → DON'T scroll. Count how many drive.google.com/thumbnail requests appear. Then scroll down. Count again.
    Initial count is small (only above-the-fold thumbnails). Count grows as you scroll. Confirms loading="lazy" is doing its job.
♿ Phase 9: Accessibility & SEO (added April 26)
Verifies the Open Graph / Twitter Card meta, ARIA labels, alt text, focus rings, and reduced-motion support shipped with P1-7.
  • Open Graph preview renders on social
    Paste https://gilinaveh.github.io/yoav-memorial/ into a WhatsApp chat (to yourself), Slack DM, or use opengraph.xyz.

  • Tab through the page with the keyboard
    Click somewhere on the page, then press Tab repeatedly. Watch for focus rings.
    A gold outline appears around each focusable element (nav links, login button, language toggle, comment fields, etc.). Mouse clicks should NOT show the ring — only keyboard focus does.
  • Screen reader announces buttons by purpose
    macOS: Cmd+F5 to toggle VoiceOver, hover the close × on the login modal.
    VoiceOver says "Close login dialog, button" — not just "button". Same for the language toggle ("Switch language") and trash icons on comments ("Delete this comment").
  • Gallery images have meaningful alt text
    DevTools → Inspector → click any photo in the gallery → look at the alt attribute on the <img>.
    Either the actual filename / caption, OR a localized fallback ("Photo of Yoav Agmon" / "תמונה של יואב אגמון") — never an empty string.
  • Reduced-motion preference is respected
    macOS: System Settings → Accessibility → Display → Reduce motion → ON. Reload the site.
    Hero stars stop twinkling, gold emblem stops pulsing, gallery images appear without the fade transition. Static memorial-friendly experience for vestibular-sensitive visitors.
💾 Phase 10: Daily Automated Backup (added April 26)
Verifies the GitHub Actions backup workflow is healthy. Run this once a month to make sure the cron is still firing and the artifacts are downloadable.
  • Latest backup run is green Critical
    Open Actions → Daily Firestore backup. Look at the most recent run.
    Most recent run timestamp is ≤ 24 hours old. Status is green ✓. If it's red or older than 24 hours, the cron has broken — re-run manually and check the FIREBASE_SERVICE_ACCOUNT secret hasn't been rotated.
  • Manual workflow_dispatch run succeeds
    Same workflow page → "Run workflow" → "Run workflow" (green button). Wait ~30 seconds.
    A new run appears at the top, completes green within ~30s. Summary block at the bottom shows the backup file size (~1-5 KB depending on data volume).
  • Artifact downloads and contains valid JSON
    Click into the run → Summary → scroll to Artifacts → download yoav-memorial-firestore-backup. Unzip → open the .json file in a text editor.
    Top of file: { "exportedAt": "...", "projectId": "yoav-memorial-7a8a3", "collections": { "candles": [...], ... } }. Counts match what's in the live Firestore.
  • Restore script can be invoked locally
    In your local terminal: FIREBASE_SERVICE_ACCOUNT="$(cat ~/path/to/key.json)" node scripts/restore-firestore.js --help (or with a real backup file pointed at a TEST collection name).
    Script runs, prints usage or restores. Don't run a full restore against production unless you actually need it — it overwrites docs.
  • Long-term backup archived monthly (manual habit)
    Set a calendar reminder for the 1st of each month: download the latest artifact and stash it in Drive / iCloud / external backup. Artifacts auto-delete after 90 days.
    You have at least 12 months of monthly snapshots somewhere off GitHub.
